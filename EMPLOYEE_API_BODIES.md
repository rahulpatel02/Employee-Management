# Employee API - JSON Request/Response Bodies

## Field Definitions

| Field | Type | Required | Description | Example |
|-------|------|----------|-------------|---------|
| `id` | Long | No* | Auto-generated unique identifier | `1` |
| `name` | String | Yes | Employee full name | `John Smith` |
| `email` | String | Yes | Unique email address | `john@company.com` |
| `department` | String | Yes | Department name | `Engineering` |
| `salary` | BigDecimal | Yes | Annual salary | `85000.00` |
| `status` | String | Yes | Employment status | `ACTIVE` or `INACTIVE` |
| `createdAt` | LocalDateTime | No* | Record creation timestamp | `2026-06-12T10:30:45` |
| `updatedAt` | LocalDateTime | No* | Last update timestamp | `2026-06-12T10:30:45` |

*Note: id, createdAt, updatedAt are auto-generated by server

---

## 1️⃣ CREATE EMPLOYEE (POST /employees)

### Request Body (201 Created)
```json
{
  "name": "John Smith",
  "email": "john.smith@company.com",
  "department": "Engineering",
  "salary": 85000.00,
  "status": "ACTIVE"
}
```

### Response Body (201 Created)
```json
{
  "id": 1,
  "name": "John Smith",
  "email": "john.smith@company.com",
  "department": "Engineering",
  "salary": 85000.00,
  "status": "ACTIVE",
  "createdAt": "2026-06-12T10:30:45.123456",
  "updatedAt": "2026-06-12T10:30:45.123456"
}
```

### cURL Example
```bash
curl -X POST http://localhost:8090/employees \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Smith",
    "email": "john.smith@company.com",
    "department": "Engineering",
    "salary": 85000.00,
    "status": "ACTIVE"
  }'
```


## 4️⃣ UPDATE EMPLOYEE (PUT /employees/{id})

### Request Body (Partial Update)
```json
{
  "name": "John Smith Updated",
  "salary": 90000.00,
  "status": "INACTIVE"
}
```

### Request Body (Full Update)
```json
{
  "id": 1,
  "name": "John Smith Updated",
  "email": "john.smith.updated@company.com",
  "department": "Engineering",
  "salary": 90000.00,
  "status": "ACTIVE"
}
```

### Response Body (200 OK)
```json
{
  "id": 1,
  "name": "John Smith Updated",
  "email": "john.smith.updated@company.com",
  "department": "Engineering",
  "salary": 90000.00,
  "status": "ACTIVE",
  "createdAt": "2026-06-12T10:30:45.123456",
  "updatedAt": "2026-06-12T10:40:15.987654"
}
```

### Error Response (400 Bad Request - Email Already Exists)
```json
{
  "status": 400,
  "message": "Email is already in use",
  "timestamp": "2026-06-12T10:40:15.987654",
  "path": "/employees/1"
}
```

### cURL Example
```bash
curl -X PUT http://localhost:8090/employees/1 \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Smith Updated",
    "salary": 90000.00,
    "status": "INACTIVE"
  }'
```

## Common Status Codes

| Code | Status | Meaning |
|------|--------|---------|
| **201** | Created | Employee successfully created (POST) |
| **200** | OK | Request successful (GET, PUT) |
| **204** | No Content | Delete successful (DELETE) |
| **400** | Bad Request | Email already exists, invalid data |
| **404** | Not Found | Employee not found by ID |
| **500** | Internal Server Error | Server error occurred |

---

## Validation Rules

- **name**: Required, max 100 characters, cannot be blank
- **email**: Required, max 100 characters, must be unique, valid email format
- **department**: Required, max 50 characters, cannot be blank
- **salary**: Required, positive decimal number (10 digits total, 2 decimal places)
- **status**: Required, max 20 characters, typically "ACTIVE" or "INACTIVE"
